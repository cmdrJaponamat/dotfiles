# dotfiles Recovery

Project path: /home/japonamat/dotfiles
Description: основной репозиторий пользовательского окружения и переносимых конфигов
Status: active

## Purpose

- Хранить переносимые пользовательские конфиги и сценарии настройки окружения.
- Быть ключевым проектом для миграции окружения на другой ПК.
- Держать под git то, что определяет поведение UI, shell, редакторов и системных helper-скриптов.

## Current State

- Репозиторий отслеживается в `~/ai/projects.db`.
- Remote переведен на `github-auto` для automation push.
- В репозитории уже есть конфиги для `niri`, `alacritty`, `fish`, `nvim`, `rofi`, `hypr`, `mako`, `ranger`, `swayidle`, `swaylock`, `wlogout`, `zathura` и сопутствующих каталогов.
- В `system/pacman-hooks/` лежит переносимый шаблон hook для пометки package inventory как stale после `pacman`-транзакций.
- Live hook для `pacman` на текущей машине уже установлен вручную через `sudo`.
- Аппаратно-зависимые части пока еще не полностью отделены от переносимой базы, это остается задачей на будущее.

## Key Commands

- Проверить состояние репозитория:
  - `git -C ~/dotfiles status`
- Отправить изменения:
  - `git -C ~/dotfiles push`
- Установить переносимый `pacman` hook на новой машине:
  - `sudo ~/dotfiles/system/pacman-hooks/install-ai-package-inventory-hook.sh`
- Проверить инвентарь системы после изменений:
  - `ai-system-inventory status`
  - `ai-system-inventory refresh`
- Зафиксировать системное изменение и протухание inventory:
  - `ai-system-change "..." "..."`
